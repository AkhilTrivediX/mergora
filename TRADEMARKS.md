# Mergora name and brand use

The MIT License applies to original Mergora code and documentation examples. It does not grant rights to imply endorsement or impersonate the official project through the Mergora name, word mark, or official visual identity.

The project does not claim a registered trademark in this policy.

You may:

- make truthful nominative references to Mergora;
- state that a product is compatible with or built using Mergora;
- create themes, integrations, forks, and educational material while clearly identifying their independent source;
- retain required attribution in redistributed project material.

You may not:

- claim to be the official Mergora project or its authorized publisher;
- use confusing package, repository, domain, organization, or account presentation that suggests official status;
- use project branding to misrepresent the provenance, evidence, maturity, or security of modified artifacts;
- imply endorsement, certification, or partnership without written permission.

Forks should use a distinct project name and explain their relationship to Mergora. Descriptive phrases such as “compatible with Mergora” should be visually secondary to the fork's own identity.
