# Support

Mergora is in bootstrap and has no Stable release or paid support offering.

## Where to ask

- Reproducible defects: use the appropriate [issue form](https://github.com/AkhilTrivediX/mergora/issues/new/choose).
- Accessibility defects: use the accessibility form; personal disability information is never required.
- Documentation defects: use the documentation form.
- Usage questions and design discussion: GitHub Discussions is the intended route once repository Discussions is enabled. Until then, open a clearly labeled issue and avoid sensitive information.
- Vulnerabilities: report privately through [SECURITY.md](SECURITY.md).
- Conduct concerns: use [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## What to include

Provide exact versions or commit, installation mode, framework/runtime, a minimal reproduction, expected and actual behavior, and sanitized logs. Do not include credentials, private source, or personal assistive-technology information.

Support is best-effort. The public compatibility and supported-version policy, once released, defines the maintained surface; a response in an issue does not create an undocumented support contract.
