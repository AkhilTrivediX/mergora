# Automated evidence

Automated evidence bundles are generated by repository commands and protected CI. Each retained record must identify:

- evidence schema version;
- full source commit and workflow SHA;
- exact tested package/registry/site digests;
- toolchain, OS, browser, locale, time zone, and relevant configuration;
- command/job, start/end time, exit result, and sanitized output digest;
- immutable artifact reference and summarized findings.

Generated records are not hand-edited. Retention policy must keep release evidence longer than ordinary diagnostic logs. A rerun does not erase an earlier failure; the disposition remains linked.

Automated records keep their source vocabulary and an explicitly validated aggregate state. Missing
runtime adapters, empty passing checks, and incomplete evidence are failures or unknown states, never
implicit skips. See [`../EVIDENCE_VOCABULARY.md`](../EVIDENCE_VOCABULARY.md).
