# Quality Passports

Passports are generated public views of a safe subset of the evidence index. Every Stable item must expose exact identity, version/commit/digest, risk, contract, automated and manual evidence, keyboard/focus behavior, responsive/input/locale/direction modes, compatibility, footprint/dependencies, Semantic Sync evidence, and limitations.

Passport states distinguish Pass, Pass with limitation, Fail, Not tested, Not applicable, and Expired. Every row requires evidence or an approved rationale. Passports are not certification badges and cannot claim universal WCAG conformance.

No Passport record is checked in until a canonical item has genuine evidence. The implemented
validators can validate Passport-context evidence but do not synthesize a Passport or passing rows.
