## Primary sources checked on 2026-07-18

- [Node.js release schedule and status](https://nodejs.org/en/about/previous-releases): Node 24 is the active LTS development/release line selected by the blueprint.
- [Next.js static export guide](https://nextjs.org/docs/app/guides/static-exports): validates `output: "export"`, build-time route generation, and the boundary against server-only features.
- [npm trusted publishing](https://docs.npmjs.com/trusted-publishers/): requires supported cloud runners, npm 11.5.1+, Node 22.14+, exact workflow/repository identity, and `id-token: write`; public GitHub-hosted publishes receive automatic provenance.
- [npm provenance statements](https://docs.npmjs.com/generating-provenance-statements/): records the public repository, hosted-runner, package metadata, and verification requirements that the prerelease gate must prove.
- [pnpm 11 settings](https://pnpm.io/settings): confirms non-auth project configuration belongs in `pnpm-workspace.yaml`; `.npmrc` is reserved for registry/auth settings. It also defines strict peer/engine checks and the explicit dependency-build allowlist used here.
- Vercel static deployment is the current website target. Revalidate the Vercel project settings,
  Node version, build command, output directory, aliases, and deployment protection before each
  release.
- [GitHub rulesets](https://docs.github.com/en/repositories/configuring-branches-and-merges-in-your-repository/managing-rulesets/about-rulesets): establishes branch/tag protection and required-check behavior.
- [React Aria ComboBox](https://react-aria.adobe.com/ComboBox) and [Modal](https://react-aria.adobe.com/Modal): establish the collection, localization, focus-containment, dismissal, and controlled-state behavior wrapped behind Mergora-owned APIs. The exact installed `react-aria-components` version is pinned in the workspace catalog.
- [TanStack Table v8 React state guide](https://tanstack.com/table/v8/docs/framework/react/guide/table-state): establishes the controlled data-model adapter boundary for the Experimental Data Grid tracer; rendered semantics and accessibility remain Mergora-owned.
- [WCAG 2.2 Recommendation](https://www.w3.org/TR/WCAG22/): the 12 December 2024 Recommendation (including errata) is the normative web-content baseline; Stable UI claims target Level AA and require full-page evidence where a page claim is made.
- [WAI-ARIA overview and version status](https://www.w3.org/WAI/standards-guidelines/aria/): WAI-ARIA 1.2 is the completed Recommendation. The 1.3 document remains a Working Draft and is monitored, not used as a conformance baseline.
- [WAI-ARIA Authoring Practices Guide](https://www.w3.org/WAI/ARIA/apg/): used as informative interaction-pattern guidance; it does not replace normative HTML, ARIA, or WCAG requirements and never proves conformance by itself.
- [U.S. Access Board Revised Section 508 standards](https://www.access-board.gov/ict/about/): establishes the federal ICT scope. The [ICT Testing Baseline for Web](https://ictbaseline.access-board.gov/web-baselines/) incorporates WCAG 2.0 Level A and AA and is tracked as a compatibility mapping beneath the stricter WCAG 2.2 AA product target.
- [ETSI EN 301 549 V3.2.1](https://www.etsi.org/deliver/etsi_en/301500_301599/301549/03.02.01_60/en_301549v030201p.pdf): the adopted edition used for the European ICT accessibility compatibility map. Later draft editions are monitored but do not replace this baseline until formally adopted.

Exact package versions were queried from the public npm registry on the same date and are locked in `pnpm-workspace.yaml` and `pnpm-lock.yaml`; mutable tags are not accepted as release evidence.
