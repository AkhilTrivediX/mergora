# Requests for comments

Numbered RFCs are required for major architecture and public API changes. An RFC states the user problem, proposed contract, accessibility/responsive/localization implications, compatibility and migration impact, security/data-loss risk, alternatives, implementation evidence, owner, and decision deadline.

Accepted RFCs produce an ADR. A proposal that conflicts with required blueprint direction must be titled `Proposed deviation` and remains unapproved until explicit approval is recorded.
