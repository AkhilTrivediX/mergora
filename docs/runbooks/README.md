# Operational runbooks

- [`INCIDENT_RESPONSE.md`](INCIDENT_RESPONSE.md): general private intake, triage, remediation, and disclosure.
- [`PARTIAL_RELEASE.md`](PARTIAL_RELEASE.md): recovery from partial npm, registry, GitHub Release, or Pages publication.
- [`COMPROMISED_PUBLISHER.md`](COMPROMISED_PUBLISHER.md): GitHub/npm identity or workflow compromise.

These are actionable starting procedures, not proof of rehearsal. Required tabletop dates, participants, observations, and follow-up issues must be added to release evidence before the operational gate can pass.
