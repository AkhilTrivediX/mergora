# Design-tool interchange

`canonical.dtcg.json` is the lossless source-preserving interchange artifact. It uses DTCG 2025.10 token objects, OKLCH colors, token aliases, compound borders/shadows/typography, and a Mergora extension that identifies the resolver and default context. Tools that implement DTCG 2025.10 should import this file together with `mergora.resolver.json`. The package places the resolver's primitive, semantic, theme, and density source files at the relative paths it declares, so the exported resolver is portable rather than a dangling index.

`design-tool-interchange.dtcg.json` is a deterministic compatibility projection for tools that cannot execute the resolver module. It is itself valid DTCG 2025.10 and contains one fully resolved group for each theme-density permutation under `workbench`; the extension maps each group name back to its theme and density context. Consumers must select exactly one group and may remove that outer group when importing it as a standalone set. Flattening preserves token paths beneath the context group, types, values, descriptions, and extensions, but intentionally does not preserve editable alias relationships between sets; the extension names `canonical.dtcg.json` as the source of truth.

The `resolved/*.dtcg.json` exports provide smaller single-context documents. No output converts OKLCH to a smaller-gamut color notation, so color information is not silently lost. A design tool that lacks OKLCH support must perform and disclose its own gamut mapping rather than treating the generated CSS fallback as canonical.
