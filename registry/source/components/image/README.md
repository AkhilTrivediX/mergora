# Image

Status: source present and unreleased. No Stable, source-security, package-parity, or manual assistive-technology claim is made.

`Image` preserves native `<img>` behavior while making informative versus decorative alternatives explicit. Its optional Mergora status rail provides visible, bounded loading, ready, failure, and source-policy rejection context; when disabled, the figcaption, live output, state callback, and related accessibility output are absent. A consumer predicate can reject a source before it reaches the native image without implying that the default predicate authenticates bytes.

Consumers own source authorization, privacy/referrer policy, optimization, CORS, caching, decoding cost, byte validation, and meaningful alternative text. Mergora does not claim to sanitize or proxy an image URL.

The literal Canvas frame, strong Ink boundary, restrained Green load mark, failure text, Violet-aware focus system for composed links, bounded radius, forced-color mapping, and motion-independent states keep this component in the family without decorating the media.
