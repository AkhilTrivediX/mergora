# Markdown

Status: source present and unreleased. No Stable, parser, sanitization, package-parity, or manual assistive-technology claim is made.

`Markdown` is intentionally a safe boundary rather than a bundled parser. With no renderer it exposes source as literal text and never uses `dangerouslySetInnerHTML`. A consumer renderer can return semantic React content after applying its own maintained parser and threat-appropriate sanitizer. The optional boundary note makes that ownership visible; the optional streaming announcer reports a short character-count summary instead of replaying a changing article. Both enhancements disappear completely when disabled.

Consumers own raw-HTML policy, URL filtering, sanitizer configuration, CSP, semantic heading/table/code output, image alternatives, and stable streaming keys. Supplying a renderer is not proof that output is sanitized.

Literal Canvas prose, Ink rules, strong heading weight, restrained Green streaming state and Violet code/context seams, bounded code surfaces, logical table alignment, shared focus geometry, narrow overflow, and forced-color mappings make rendering recognizable without decorating authored content.
