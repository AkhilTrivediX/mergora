# Status canonical source

Status: source present and unreleased. Automated workbench evidence is not release evidence; no Stable or conformance claim is made here.

Status composes a distinct symbol, visible localized variant label, and consumer text. Live behavior is opt-in.

## Contract

- live=off is the default; polite and assertive are explicit atomic modes.
- Severity meaning remains available through text and shape when color or icons are unavailable.
- Stable Provider keys are status.neutral, status.info, status.success, status.warning, status.error, and status.variantLabel. The formatter key owns punctuation and grammatical order through the `{variant}` value; an explicit variantLabel is already the complete visible cue and is never modified.
- Role and owned ARIA overrides are rejected for both typed and runtime JavaScript callers, and empty composite content is invalid.

The public ref and stable data-slot parts are recorded in status.anatomy.json. The exact five-key source manifest, metadata, API, story-state policy, accessibility contract, and honest promotion delta live beside the implementation.

## Promotion boundary

Generation drift, strict types, unit, SSR, hydration, schema, browser, axe, announcement and preference gates, packed consumers, package/source parity, Semantic Sync fixtures, manual assistive-technology sessions, reviewed visual evidence, public-site dogfooding, and an approved digest-bound Quality Passport remain required.
