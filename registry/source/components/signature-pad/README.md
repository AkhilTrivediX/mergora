# Signature Pad

Status: source present and unreleased. No Stable, legal-validity, biometric, file-safety, package-parity, or manual Risk Class 3 claim is made.

`SignaturePad` stores normalized vector strokes instead of making canvas pixels the only value. The canvas accepts pointer and keyboard input with RTL-aware cursor movement; undo and clear remain explicit. Disabled-aware form serialization, reset, and required-state recovery remain native-form compatible. Visible keyboard controls, typed input, native file input, polite change summaries, and a legal caveat are independent options. Disabling any option removes its controls, callbacks, state, and accessibility output.

The component is not identity proof, a biometric system, a legal-signature service, or a file scanner. Consumers own consent, signer identity and intent, jurisdictional requirements, audit/retention, validation, authorization, server storage, and a non-canvas path appropriate to the task.

A literal white signing surface, strong Ink frame, decisive label, Green selected state, Violet keyboard rail, shared focus geometry, bounded corners, logical direction, wrap-safe controls, forced-color mappings, and motion-independent drawing keep the experience coherent without ceremonial decoration.
