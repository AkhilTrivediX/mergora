# Calendar canonical source

Status: source present and unreleased. No Stable, package-parity, conformance, or manual assistive-technology claim is made.

`Calendar` combines a localized keyboard grid with a real `input type="date"`, so typed entry, mobile pickers, validation, serialization, autofill, and reset remain native. It supports controlled and uncontrolled canonical `YYYY-MM-DD` values, locale weekday/month labels, configurable week starts, bounds, unavailable dates, month/year keyboard movement, RTL-aware horizontal arrows, and responsive touch targets. Optional `highlightRange` and `onDatePreviewChange` provide a bounded composition seam for range systems; when omitted, no range attributes, pointer events, blur events, or preview output are added.

The Mergora advantage is optional blocked-date explanation. `showAvailabilityExplanations` defaults to `false`; when false it creates no reason node or described-by relationship. When true, only unavailable dates with consumer-supplied reasons receive associated screen-reader text. The literal Canvas surface, Ink boundary, Green selection, Violet focus seam, strong heading, and restrained radii use shared semantic tokens.

Promotion remains blocked on regenerated package/source/Shadcn parity, packed consumers, broad browser evidence, and current manual screen-reader, speech, switch, and mobile assistive-technology records.
