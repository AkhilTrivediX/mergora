# Attachment

Status: source present and unreleased. No Stable, file-safety, transport, package-parity, or manual assistive-technology claim is made.

`Attachment` presents durable file identity, type, locale-formatted size, consumer-owned lifecycle, and explicit preview/download/remove actions. Its Mergora advantage is an optional safety rail that distinguishes application-verified, unverified, and unsafe states without implying that a filename or MIME string proves safety. Preview, status, safety guidance, and checksum output are independently disabled by default; disabling one removes its UI, live output, callbacks, and related semantics.

The component never opens, reads, scans, uploads, downloads, persists, or deletes file bytes. Consumers must validate file signatures and content, authorize access, construct safe preview/download behavior, confirm destructive removal when necessary, and keep errors outside transient-only UI.

The family signature uses a literal Canvas surface, Ink rules, decisive file naming, a Living Green lifecycle signal, a restrained Violet integrity cue, bounded shared radii, logical layout, shared focus geometry, forced-color mappings, and no decorative motion.
