# DateRangePicker canonical source

Status: source present and unreleased. No Stable or parity claim is made.

`DateRangePicker` coordinates two labelled native date controls, controlled or uncontrolled canonical range state, bounds, required validation, reset, and form serialization. Consumer-owned presets, inclusive duration summaries, and inclusive `minimumDurationDays` / `maximumDurationDays` enforcement are separate opt-in enhancements. Duration bounds derive native end constraints, set recoverable custom validity and associated error text, and can report typed issues through `onDurationIssueChange`; omitting both bounds removes that validation UI, event, and accessibility path without changing the native ordered range.

Mergora Ink structure, strong labels, Canvas surfaces, Green duration rail, explicit danger recovery, Violet focus treatment, responsive collapse, and semantic tokens keep the system coherent. Promotion remains blocked on generated parity, packed consumers, the full environment matrix, and manual assistive-technology records.
