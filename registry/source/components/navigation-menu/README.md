# NavigationMenu canonical source

Status: source present and unreleased. No Stable, package-parity, or manual assistive-technology claim is made.

`NavigationMenu` is site navigation, not an application menu: it uses a named `nav`, lists, disclosure buttons, and real links without menu roles or arrow-key traps. One controlled or uncontrolled group opens at a time; Escape closes its panel and restores trigger focus. Narrow layouts keep the same semantics while making panels in-flow.

The optional destination preview runs only for focused or pointer-entered enabled links inside an open destination group, where the preview can remain adjacent to its source link. Omitting `renderLinkPreview` removes its handlers, preview state, region, name, and output. Literal Canvas panels, Ink boundaries, strong type, Green current state, Violet focus geometry, restrained corners, RTL-safe logical placement, and forced-color mappings repeat the Mergora signature.

Promotion still requires generated parity, packed consumers, cross-engine evidence, and current manual keyboard, screen-reader, speech, switch, and touch records.
