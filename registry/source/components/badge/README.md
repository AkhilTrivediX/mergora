# Badge canonical source

Status: source present and unreleased. Automated workbench evidence is not release evidence; no Stable or conformance claim is made here.

Badge is a strictly noninteractive category, status, or count token. A badge never turns into a button or link.

## Mergora signature and advantage

The compact squared label, Ink boundary, readable typography, and restrained semantic fills align Badge with the Mergora feedback family without turning every label into a pill. Beyond a baseline Shadcn Badge, explicit status mode localizes severity and count mode keeps the exact localized count accessible while visually bounding large values. The default `kind="category"` enables neither enhancement and therefore emits no severity prefix, hidden exact-count text, overflow state, or count formatting work.

## Contract

- Every React event handler plus href, accessKey, editable/drag state, focus, role, dangerous HTML, and generic accessible-name overrides are rejected by TypeScript and runtime; wrap Badge in a named Button or Link when interaction is required.
- Status mode exposes visible localized severity without truncation. `badge.status` formats its punctuation and the order of severity and label; long labels wrap and remain readable.
- Count mode formats locale-aware numbers, visually caps at maximum, and keeps the exact count in its accessible label.
- Stable Provider keys are badge.neutral, badge.info, badge.success, badge.warning, badge.error, badge.status, and badge.count.

The public ref and stable data-slot parts are recorded in badge.anatomy.json. The exact five-key source manifest, metadata, API, story-state policy, accessibility contract, and honest promotion delta live beside the implementation.
