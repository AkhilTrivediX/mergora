# Navbar canonical source

Status: source present and unreleased. No Stable, package-parity, or manual assistive-technology claim is made.

`Navbar` composes a brand region, skip link, real desktop links, a focus-returning mobile disclosure, current-page state, disabled non-links, Escape dismissal, and consumer-controlled router interception. Mobile and desktop output use CSS visibility so only the relevant navigation landmark participates at each breakpoint.

An optional router-owned status rail adds loading or failure context without coupling the component to a framework. Omitting `routeStatus` removes the output, live-region behavior, `aria-busy`, and status attributes. The visual language uses literal Canvas, decisive Ink rules, restrained Green selection, Violet focus geometry, narrow reflow, forced-color mappings, and no decorative elevation.

Promotion still requires generated parity, packed consumers, full responsive/browser evidence, and current manual assistive-technology sessions.
